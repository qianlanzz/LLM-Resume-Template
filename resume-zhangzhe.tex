\sectionTitle{比赛经历}{\faTrophy}
\begin{itemize}
  \item \textbf{KDD Cup 2024 学术论文问答检索挑战赛} \hfill 2024.05 -- 2024.07
  \begin{itemize}
    \item \textbf{任务背景}: 面向学术问答场景，从海量论文中检索与专业问题最匹配的候选论文，核心挑战在于复杂 Query 冗长、语义匹配困难。
    \item \textbf{方案设计}: 设计``问题精炼 + 向量召回 + 重排序''两阶段检索方案——基于 Llama3 对 Query 精炼总结，使用 BGE M3 统一向量化，结合 FAISS 召回 Top100 候选，再引入 BGE-Reranker 精排筛选 Top20；通过 Hard Example Mining 构建微调数据，对 Reranker 进行专项优化，提升困难样本区分能力。
    \item \textbf{最终成果}: 官方测试集检索分数从 0.138 提升至 0.166，较基线提升约 20.3\%。
  \end{itemize}

  \item \textbf{``天翼云息壤杯'' AI 大赛：大语言模型数学推理能力提升} \hfill 2024.11 -- 2025.07
  \begin{itemize}
    \item \textbf{任务背景}: 基于 Qwen3-8B 开展微调训练，提升模型对复杂数学问题与深度推理的理解求解能力。
    \item \textbf{方案设计}: 针对数学推理长链路推导困难、监督样本质量不足的问题，设计``数据蒸馏 + 质量筛选 + 参数高效微调''整体方案——基于 MetaMathQA 构建数据集，利用 Qwen2.5-Math-72B 蒸馏生成 10 万条高质量 CoT 样本，结合 Math-Verify 与 Qwen2.5-72B 评审模型做答案校验。
    \item \textbf{工程落地}: 在 8 卡昇腾 910B 64G 环境完成 Qwen3-8B 的 LoRA 微调，结合 vLLM 优化推理性能，实现训推一体化部署。
    \item \textbf{最终成果}: 微调后客观数学题正确率从 38\% 提升至 62\%，提升幅度达 63\%。
  \end{itemize}
\end{itemize}

\sectionTitle{项目经历}{\faCode}
\begin{itemize}
  \item \textbf{AgentBot · LLM Agent 框架（开源个人项目）} \hfill 2025.10 -- 至今
  \begin{itemize}
    \item 基于 LangGraph StateGraph 搭建的 CLI Agent，核心是 agent $\leftrightarrow$ tools 循环工作流，支持多轮工具调用与条件路由；通过统一的 Provider 工厂适配 OpenAI / Anthropic / 阿里云等 6 家模型 API，运行时可热切换。
    \item 记忆分两层：长期记忆用 Markdown 文件存用户偏好，LLM 通过 Tool Call 自己决定什么时候更新，跨会话保留；短期记忆基于 SQLite Checkpointer 存完整对话，按回合切分，超 40 轮后自动让 LLM 把旧对话压缩成摘要再裁掉原文，长对话 Token 用量降低约 60\%。
    \item 内置 12 个工具（时间、计算器、任务调度、文件读写、Shell 执行等），工具调用走 LangChain @tool 标准接口；文件和 Shell 操作限制在沙盒目录内，通过路径前缀校验拦截穿越，正则匹配拦截 rm -rf 等危险命令，Shell 执行设 60s 超时。
    \item 技能（Skill）是可热插拔的外部扩展，放一个带 SKILL.md 的目录就能自动扫描注册为 LangChain Tool；执行前 LLM 先读 SKILL.md 了解用法和副作用再决定是否调用，自建 20 组评测集验证该机制将误操作率从 50\% 降至 10\%；同时支持通过 MCP 协议接入外部工具服务。
    \item 所有 LLM 输入、工具调用、工具返回、AI 回复、系统动作共 5 类事件写入 JSONL 审计日志（单例 + 内存队列 + 守护线程异步落盘），配套 Rich 终端可实时查看 Agent 每一步的决策过程。
  \end{itemize}

  \item \textbf{教辅录题审核一体化系统} \hfill 2025.06 -- 至今
  \begin{itemize}
    \item \textbf{项目背景}: 原有教辅录题依赖 Excel + 多工具协作，流程碎片化、效率低，需要一套打通上传、录题、质检、入库全链路的一体化系统。
    \item \textbf{主流程建设}: 基于 Vue 3 + TypeScript、Fastify + Prisma + PostgreSQL + Redis 开发录题工作台及后端服务，支持 PDF 双栏浏览、框选 OCR、公式渲染、草稿自动保存，打通教材上传到结果入库的完整链路。
    \item \textbf{结构化自动切题}: 主导自动切题模块，基于 MinerU 的块级坐标解析与目录分段能力将教材 PDF 转换为结构化内容，结合大模型完成题号、题干、选项、答案、解析抽取，基于``章节标题 + 题号''实现题答配对，生成包含编号、页码、坐标、题型等字段的标注结果回写工作台。
    \item \textbf{稳定性治理}: 基于 BullMQ 承接 PDF 拆页、MinerU 解析与自动切题等长任务，避免重任务阻塞工作台；补充队列监控告警、草稿缓存与历史恢复、提交前校验等机制，降低卡顿、丢稿和返工风险。
  \end{itemize}
\end{itemize}
